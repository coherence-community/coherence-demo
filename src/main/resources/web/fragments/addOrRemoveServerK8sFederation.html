Since the demonstration was started using Kubernetes, you must use the Kubernetes command kubectl apply to scale the application rather than specifically adding or removing members.

You have also configured Federation, so you must use the following to scale the Primary and Secondary clusters.

Primary

To scale the number of replicas to 2, update the primary-cluster.yaml used to create the cluster and change the replicas to 2 under the storage role: 

  replicas: 2
Then apply this change using 
$ kubectl apply --namespace coherence-example -f yaml/primary-cluster.yaml

Secondary

To scale the number of replicas to 2, update the secondary-cluster.yaml used to create the cluster and change the replicas to 2 under the storage role: 

  replicas: 2
Then apply this change using 
$ kubectl apply --namespace coherence-example -f yaml/secondary-cluster.yaml
You can also scale the cluster using the following for the primary-cluster. 
$ kubectl scale --namespace coherence-example coherence/primary-cluster-storage --replicas=6
