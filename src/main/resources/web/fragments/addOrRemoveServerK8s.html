Since the demonstration was started using Kubernetes, you must use the Kubernetes command kubectl apply to scale the application rather than specifically adding or removing members.


To scale the number of replicas to 2, update the primary-cluster.yaml used to create the cluster and change the replicas to 2 under the storage role: 

  replicas: 2
Then apply this change using 
$ kubectl apply --namespace coherence-example -f yaml/demo-cluster.yaml

You can also scale the cluster using 

$ kubectl scale coherence --namespace coherence-example primary-cluster-storage --replicas=6

You can use kubectl get pods to view the status of the apply command and the number of replicas.

If you are scaling down, this will be done safely to ensure you do not lose any data.
