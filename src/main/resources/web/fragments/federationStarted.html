A secondary Coherence cluster has been started and can be accessed via the "Federation" menu.

The federated caching feature federates cache data asynchronously across multiple geographically dispersed clusters. Cached data is federated across clusters to provide redundancy, off-site backup, and multiple points of access for application users in different geographical locations.

Once Federation is started any updates to data on either the primary or secondary clusters are replicated to the other. The default topology, if none is specified, is Active-Active.

To configure a Federated cache, change your distributed-scheme to a federated-scheme in your cache configuration. 

  <federated-scheme>
    <scheme-name>federated-scheme</scheme-name>
    ...
    <autostart>true</autostart>
  </federated-scheme>
You must also add the clusters into your operational configuration in the federation-config section. 
  <federation-config>
    <participants>
      <participant>
        <name>PrimaryCluster</name>
        <initial-action>start</initial-action>
        <name-service-addresses>
          <socket-address>
            <address>127.0.0.1</address>
            <port>7574</port>
          </socket-address>
        </name-service-addresses>
      </participant>
      <participant>
        <name>SecondaryCluster</name>
        <initial-action>pause</initial-action>
        <name-service-addresses>
          <socket-address>
            <address>127.0.0.1</address>
            <port>7575</port>
          </socket-address>
        </name-service-addresses>
      </participant>
    </participants>
  </federation-config>
For more information of Federation see the Oracle Documentation.
